Working Environment

You will be using an Subversion (SVN) repository to manage your assignments. Your life will be considerably easier if you read and follow the instructions for working on and submitting your assignment. If you need a refresher on how to use Subversion, please refer to the Version Control with Subversion book. Appendix A is a quick start guide and a good place to start.

If you are comfortable using Git, a distributed version control system, or would like to learn more about it, you may be interested in trying git-svn, which is a mechanism for using Git locally while interfacing with remote Subversion repositories. It is still your responsibility to ensure that your work is correctly committed to the supplied SVN repository, and any use or misuse of git-svn will not be considered as a valid excuse for incorrect assignment submissions.
